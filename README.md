# LoongEnglish

> 用中文的组合能力，重写英语的表达方式；再借助 AI 扩散新词、新结构、新语感，逐步把“中式英语”从偏见标签，变成一种真正可用、可传播、可进化的新语言方案。

## 项目简介

`LoongEnglish` 是一个带有实验性质的语言工程项目。

这个项目的出发点很简单：英语经过长期历史叠加，存在大量不规则、低透明度、强记忆负担的表达方式。对于中文使用者来说，英语里很多词汇构词逻辑并不直观，很多固定搭配也缺乏可推理性，学习成本高，迁移成本高，表达门槛也高。

相反，中文在组合、压缩、造词、语义嫁接方面具有非常强的工程优势。中文天然擅长把复杂概念拆成若干个更稳定、更直觉的语义单元，再通过组合形成新表达。这种能力，本质上是一种高效率的“语义模块化能力”。

所以这个项目提出一个激进但认真对待的设想：

**既然英语本身已经越来越像一套历史包袱极重的遗留系统，那我们就不再被动适应它，而是尝试用中文的优良组合特性去改造它，形成一套更适合中文思维、更容易推理、更容易生成、更适合 AI 协作传播的中式英语。**

我们的目标不是做几句玩笑式的 Chinglish，而是：

- 建立一套可解释的造词规则
- 建立一套可扩展的语义组合体系
- 建立一套 AI 可学习、可复述、可生成、可推广的表达协议
- 最终让“中式英语”从一种使用偏差，变成一种拥有自身规范、词库和传播能力的新表达系统

如果这套系统足够高效、足够清晰、足够有感染力，它就不只是辅助表达工具，而可能反过来影响真实世界里的英语使用习惯。

## 核心判断

这个项目建立在以下几个判断之上：

### 1. 语言并不神圣，语言是可以被工程化改造的

语言不是只能被继承的东西，也可以被重新设计、重新压缩、重新分发。只要一套表达系统在沟通效率、记忆成本、传播速度上更优，它就有机会获得真实使用场景。

### 2. 英语的全球地位，并不意味着它的结构就是最优的

英语的优势更多来自历史、殖民、工业体系、互联网和学术话语权，而不一定来自其内部结构的优雅程度。很多不规则词形、固定搭配和例外规则，本质上都增加了学习成本。

### 3. 中文思维适合做“二次语言设计”

中文用户长期习惯以“字义拼装”“语义并列”“上下文补全”的方式组织语言，这使得中文天然适合参与新词系统设计。中文不是英语的次级适配器，中文完全可以反过来成为一种“英语重构引擎”。

### 4. AI 会成为新语言扩散的最大放大器

过去新词扩散依赖社群、媒体和代际传播；现在，AI 和大模型已经成为新的语言中介层。只要一套新词和新组合能被 prompt、语料、工具链、Agent、翻译器和内容生成系统持续使用，它就可能被迅速放大。

## 项目目标

`LoongEnglish` 希望逐步完成四件事：

1. 定义一套中式英语的基本构词原则和表达规范
2. 沉淀一批高频概念的新造词、新短语和新句法模板
3. 用 AI 工具链让这些表达具备自动生成、自动解释、自动翻译、自动传播能力
4. 形成一个可被社区协作维护、持续演化的开放语言实验

## 方法论

这个项目不会简单地把中文逐字翻成英语，而是会围绕以下方向持续实验：

### 1. 组合优先

优先使用可拆解、可推理、可复用的语义单元，而不是背诵式固定词块。

### 2. 透明优先

一个新词是否成立，首先看它能不能让第一次见到的人大致猜到意思。

### 3. 高频优先

优先改造日常交流、知识表达、互联网内容、产品协作中最常见、最痛苦、最难记的部分。

### 4. AI 友好优先

新表达不仅要让人能理解，也要让模型容易学习、容易迁移、容易稳定输出。

### 5. 词根继承优先

核心语义词根，优先从现有英语中抽取，而不是直接拼音化；组合顺序，则优先保留中文结构。也就是说，`LoongEnglish` 更接近“英语词根 + 中文组合”，而不是简单的拼音英语。

### 6. 社区迭代优先

任何真正能活下来的新语言，都不是一次性设计出来的，而是在大量使用、纠错、模仿和再创造中长出来的。

## 分阶段路线图

### 第一阶段：立项与规则定义

目标：先把“中式英语到底是什么”说清楚。

这一阶段主要完成：

- 明确项目宣言、目标边界和核心态度
- 定义基本造词原则、语义组合规则、命名偏好和表达风格
- 梳理英语中最不透明、最不规则、最不利于中文使用者掌握的区域
- 建立第一版基础词库框架
- 输出示例集，让外部使用者能快速理解项目方向

阶段产物：

- `manifesto` 项目宣言
- `spec` 构词与表达规范 v0.1
- `seed lexicon` 种子词库
- `examples` 对照示例集

### 第二阶段：词库建设与表达实验

目标：从抽象设想进入具体造词和句型实验。

这一阶段主要完成：

- 建立高频词汇替代方案
- 为常见概念设计更符合中文思维的新表达
- 沉淀常用短语、句型模板和场景化表达
- 记录每个新词的语义来源、使用场景、替代优势和争议点
- 建立“原英语 / 中式英语 / 中文解释”的三向对照体系

重点场景：

- 日常交流
- 互联网表达
- 产品与技术协作
- 学习与知识传播
- AI 提示词与 Agent 指令

阶段产物：

- 高频词库 v1
- 常用表达手册 v1
- 场景化示例库
- 造词提案机制

### 第三阶段：AI 接入与自动化扩散

目标：不只人工写词库，而是让模型参与生成、筛选、翻译、推广。

这一阶段主要完成：

- 训练或微调一套中式英语生成提示模板
- 构建词汇推荐、改写、对照翻译、语义解释等能力
- 让模型在聊天、写作、内容生成中优先尝试中式英语表达
- 设计评估机制，观察哪些词最容易被理解、复用和传播
- 建立机器可读的词典、规则表和 prompt 资产

阶段产物：

- AI 生成提示词库
- 中式英语转换器原型
- 机器可读词典格式
- 用法分析与传播数据面板

### 第四阶段：社区传播与共创规范

目标：让这套语言开始脱离个人设想，进入集体协作。

这一阶段主要完成：

- 建立社区投稿、评审、投票和版本发布机制
- 鼓励用户提交新词、例句、梗图、视频、文章和教程
- 形成统一的词条结构和贡献规范
- 在社交媒体、内容平台、开发者社区和 AI 社区进行扩散实验
- 逐步建立“哪些表达已经被广泛接受”的事实标准

阶段产物：

- 贡献指南
- 社区词条评审机制
- 月度版本发布制度
- 内容传播模板

### 第五阶段：标准化与反向影响

目标：让中式英语从“实验项目”进入“可持续语言协议”。

这一阶段主要完成：

- 将成熟表达沉淀为稳定规范
- 形成公开可引用的词典、白皮书和语言设计文档
- 推动在教育、内容创作、产品命名、AI 系统提示中使用
- 尝试让模型默认支持、理解并输出部分中式英语表达
- 观察其是否对真实英语使用生态产生反向影响

理想结果：

- 一部分新词进入稳定使用
- 一部分组合方式被广泛模仿
- 一部分表达进入 AI 默认语料习惯
- “中式英语”不再只是被纠正，而是开始被讨论、被引用、被借用

## 执行计划

为了让这个项目真正推进，而不是停留在概念层，当前执行计划分为三个时间尺度。

### 短期计划（1 到 2 个月）

- 完成 README、宣言和规则草案
- 整理首批 100 到 300 个高频概念词条
- 为每个词条补充中文解释、原英语映射和新表达建议
- 挑选 20 到 50 组代表性例句进行改写实验
- 建立最小可用的数据结构，便于后续被 AI 工具读取

### 中期计划（3 到 6 个月）

- 扩充词库到 1000+ 条
- 完成若干典型场景下的表达手册
- 做出一个基础转换器或 prompt-based assistant
- 引入社区贡献和评审机制
- 用真实内容平台测试传播效果与理解成本

### 长期计划（6 到 24 个月）

- 沉淀相对稳定的版本体系
- 建立官网、在线词典或交互式演示工具
- 构建与大模型联动的 API / Agent / 插件能力
- 与教育、写作、产品命名、内容创作等场景结合
- 让中式英语成为一种可被持续生产和持续消费的表达生态

## 版本原则

项目初期遵循以下版本原则：

- `v0.x`：以探索、试错、快速迭代为主
- `v1.0`：形成基础规范、核心词库和公开协作流程
- `v2.0`：形成 AI 工具链与较稳定的扩散机制
- `v3.0`：具备跨社区传播能力，并对主流表达产生可观测影响

## 项目价值

如果这个项目成立，它的价值不只是“造一些有趣的新词”，而是：

- 降低中文用户进入英语表达系统的认知成本
- 把中文思维从被动翻译，升级为主动造词
- 让 AI 不只是纠正中式英语，而是参与定义中式英语
- 把语言学习从服从既有规则，变成共同设计新规则

从更长远的角度看，`LoongEnglish` 不是单纯要学会英语，而是试图争夺一部分英语的定义权。

## 当前状态

项目当前仍处于 `v0.1` 概念定义阶段，优先事项包括：

- 写清楚核心愿景和边界
- 建立第一版规则草案
- 准备第一批词条样本
- 设计适合 AI 使用的数据组织方式
